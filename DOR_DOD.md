# ✅ DoR (Definition of Ready) e ✅ DoD (Definition of Done)

Este documento descreve os critérios padrão de Definition of Ready (DoR) e Definition of Done (DoD) usados neste repositório. Está escrito em Português (PT-BR).

## ✅ Definition of Ready (DoR)
- Descrição clara do objetivo/escopo da tarefa.
- Critérios de aceitação definidos e mensuráveis.
- Dependências identificadas e documentadas (APIs externas, bibliotecas, infraestrutura).
- Estimativa de esforço (story points / horas) atribuída.
- Ambiente de desenvolvimento e acesso configurados.
- Mockups ou designs anexados quando aplicável.
- Pessoa responsável (assignee) definida.

## ✅ Definition of Done (DoD)
- Código implementado seguindo as convenções do repositório.
- Testes automatizados adicionados (unitários/integrados) cobrindo casos relevantes.
- Todos os testes passam localmente e no CI.
- Código revisado por pelo menos um revisor (PR aprovado).
- Documentação atualizada (README, changelog ou docs específicos).
- Não há warnings relevantes no build.
- Migrações e alterações de banco de dados documentadas e testadas.
- Deploy realizado / instruções de deploy descritas quando aplicável.

## Observações
- Use este checklist como referência e adapte conforme a complexidade da tarefa.
- Se algum critério não for aplicável, documente o motivo no PR.
